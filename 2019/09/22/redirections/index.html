文件描述符(file-descriptor)

  • 定义解释
    • 文件描述符是一个数字, 用于唯一标识计算机操作系统中打开的文件
    • 它描述了数据资源, 以及如何访问该资源
  • 场景
    • 当程序需要打开一个文件或资源的时候
    • 操作系统内核授予访问权限, 在全局文件表中创建一个条目
    • 反馈给程序相应的条目地址
  • 碎片式理解
    • 文件描述符是文件或输入/输出资源的抽象标识, 被简写为fd
    • 存在于unix-like的操作系统之中
    • 也被应用于windows上, 被称为file handle句柄
    • 文件描述符用一个非负的整数表述, 如 0, 1, 2, 255
    • 每个系统进程, 都将默认分配0, 1, 2, 三个文件描述符
      • 具体分配
        fd Name Abbreviation Description Example
        0 Standard input stdin 默认标准输入 pipline(|), 键盘输入
        1 Standard output stdont 默认标准输出 print命令, 键盘显示
        2 Standard error stderr 默认标准错误输出 命令不存在提示, 屏幕显示

正文内容

输入重定向

  • < 是输出重定向的标志
  • 输入重定向, 将需要从键盘获得的信息通过重定向由文件提供
  • 应用场景: 发送邮件

输出重定向

  • > 是输出重定向的标志

  • 输出重定向, 将应用/程序运行的结果重定向到文件/资源上

  • 应用场景: 过滤有效信息, 保存到文件中

  • 应用场景: 过滤有效信息, 显示在屏幕同时保存到文件中 (tee)

输出追加重定向

  • >> 是输出追加重定向的标志
  • 输出重定向>, 会覆盖文件内容, 重定向的文件保存上一次输出内容
  • 输出追加重定向>>, 将会在原有文件内容尾部添加内容, 重定向文件保存每一次输出内容
  • 应用场景: 过滤多个文件有效信息, 保存到文件中

标准输出/错误输出

  • 系统默认规定, 1为标准输出, 2为标准错误输出

  • 是否是错误输出的标准, 可在命令执行后, 通过$?进行判断

    • $? –> 0 –> 正常运行
    • $? –> 非0 –> 错误运行

  • 标准输出, 使用>或1>进行输出重定向 (1可省略)

  • 标准错误输出, 使用2>进行输出重定向 (2不可省略)

  • 当运行脚本时, 非脚本提示信息, 运行中的标准输出/错误输出应被屏蔽掉

    • ____ 需要填写重定向信息
    • 是否通信正常是无法确认的, 输出可能是标准输出或标准错误输出
    • 但这两种输出信息我们都不希望打印出来, 需要对其均定向为空(/dev/null)

  • 方法一: (分离重定向)

    • 将标准输出和标准错误分别重定向

  • 方法二: (同时重定向)

    • 通过 &> 或 >& 重定向

  • 方法三: (文件标识符转移)

    • 将标准输出定向到空, 后将标准错误定向到标准输出

    • 需要注意这个命令有顺序之分, 切勿写成

    • 这样只有标准输出会被重定向到空中, 标准错误将在屏幕上显示

    • 用来产生一个新进程
    • 这个进程默认会复制自身
  • exec

    • 不配合fork时:
      • 结束当前进程, 执行指定进程
    • 配合fork时:
      • 在当前进程启动另一个进程

cheat sheet

Redirection Description
cmd > file Redirect the standard output (stdout) of cmd to a file.
cmd 1> file Same as cmd > file. 1 is the default file descriptor (fd) for stdout.
cmd 2> file Redirect the standard error (stderr) of cmd to a file.
2 is the default fd for stderr.
cmd >> file Append stdout of cmd to a file.
cmd 2>> file Append stderr of cmd to a file.
cmd &> file Redirect stdout and stderr of cmd to a file.
cmd > file 2>&1 Another way to redirect both stdout and stderr of
cmd to a file. This is not the same
as cmd 2>&1 > file. Redirection order matters!
cmd > /dev/null Discard stdout of cmd.
cmd 2> /dev/null Discard stderr of cmd.
cmd &> /dev/null Discard stdout and stderr of cmd.
cmd < file Redirect the contents of the file to the standard input (stdin) of cmd
cmd << EOL
line1
line2
EOL		 Redirect a bunch of lines
to the stdin. If ‘EOL’ is quoted, text is treated literally. This is called a here-document
cmd <<- EOL
foo
bar
EOL		 Redirect a bunch of lines to the stdin and strip the leading tabs.
cmd <<< “string” Redirect a single line of text to the stdin of cmd.
This is called a here-string.
exec 2> file Redirect stderr of all commands to a file forever.
exec 3< file Open a file for reading using a custom file descriptor.
exec 3> file Open a file for writing using a custom file descriptor.
exec 3<> file Open a file for reading and writing using a custom file descriptor.
exec 3>&- Close a file descriptor.
exec 4>&3 Make file descriptor 4 to be a copy of file descriptor 3. (Copy fd 3 to 4.)
exec 4>&3- Copy file descriptor 3 to 4 and close file descriptor 3.
echo “foo” >&3 Write to a custom file descriptor.
cat <&3 Read from a custom file descriptor.
(cmd1; cmd2) > file Redirect stdout from multiple commands to a file
(using a sub-shell).
{ cmd1; cmd2; } > file Redirect stdout from multiple commands to a file
(faster; not using a sub-shell).
exec 3<> /dev/tcp/host/port Open a TCP connection to host:port.
(This is a bash feature, not Linux feature).
exec 3<> /dev/udp/host/port Open a UDP connection to host:port.
(This is a bash feature, not Linux feature).
cmd <(cmd1) Redirect stdout of cmd1 to an anonymous fifo,
then pass the fifo to cmd as an argument. Useful when cmd doesn’t read from stdin directly.
cmd < <(cmd1) Redirect stdout of cmd1 to an anonymous fifo,
then redirect the fifo to stdin of cmd. Best example: diff <(find /path1
cmd <(cmd1) <(cmd2) Redirect stdout of cmd1 and cmd2 to
two anonymous fifos, then pass both fifos as arguments to cmd.
cmd1 >(cmd2) Run cmd2 with its stdin connected to an anonymous fifo,
and pass the filename of the pipe as an argument to cmd1.
cmd1 > >(cmd2) Run cmd2 with its stdin connected to an anonymous fifo,
then redirect stdout of cmd to this anonymous pipe.
cmd1 | cmd2 Redirect stdout of cmd1 to stdin of cmd2. Pro-tip:
This is the same as cmd1 > >(cmd2),same as cmd2 < <(cmd1),
same as > >(cmd2) cmd1, same as < <(cmd1) cmd2.
cmd1 |& cmd2 Redirect stdout and stderr of cmd1 to stdin of cmd2 (bash 4.0+ only)
Use cmd1 2>&1
cmd | tee file Redirect stdout of cmd to a file and print it to screen.
exec {filew}> file Open a file for writing using a named file descriptor
called {filew} (bash 4.1+).
cmd 3>&1 1>&2 2>&3 Swap stdout and stderr of cmd.
cmd > >(cmd1) 2> >(cmd2) Send stdout of cmd to cmd1 and stderr of cmd to cmd2.
cmd1 | cmd2 | cmd3 | cmd4
echo ${PIPESTATUS[@]}		 Find out the exit codes of all piped commands.
