处理文件

工作原理

  1. 从文件从读取一行, 放入pattern-space –> Read
  2. 执行处理指令 –> Execute
  3. 打印pattern-space内容 –> Print
  4. 重复1-3步骤, 如读取到文件末尾, 终止退出 –> Repeat

基本语法

基本命令

打印(p 命令)

  • p命令 用于打印出pattern-space的内容

  • 由于sed默认打印出pattern-space空间内容

  • 直接使用p命令, 将会得到二次打印的结果

  • -n选项能关闭sed默认打印的功能

  • 因此p命令常与-n选项结合使用

地址范围定义

行范围定义 (,)

自定义地址 (+, ~, //)

删除 (d 命令)

  • 所谓删除, 意味着不再输出中显示
  • 原文件内容并不会被删除
  • d命令与p命令类似, 均可以与地址匹配一起使用

写入文件 (w 命令)

  • 想对于将pattern-space中内容直接输出
  • w命令可以将其, 输出到指定文件中
  • (虽然许多人可能喜欢使用重定向解决 但sed的确提供了这个功能)

替换命令 (s)

基本语法

示例说明

替换标记

全局替换 (g 标记)

  • 默认情况下, s命令只对本行第一次匹配到的做替换
  • 如果需要对本行所有内容都进行替换操作, 需要添加g标记

数字标记 (1, 2, 3.. 标记)

  • sed默认匹配每行第一次出现的字符串
  • 如需指明替换的字符串在第几次出现,可使用数字标记

打印 (p 标记)

  • 和p命令类似, 打印出pattern-space处理后的结果

写入 (w 标记)

  • 和w命令类似 保存pattern-space到文件中

忽略大小写 (i 标记)

  • sed默认对匹配字符串大小写敏感
  • i标记可以取消大小写敏感

命令执行 (e 标记)

  • 将pattern-space中的内容当作命令执行

标记组合

  • 标记之间可以组合使用

自定义分割符

  • sed 默认的替换分割符为(/)
  • 当碰到路径替换时, 需要使用(\), 将会显得十分臃肿难看
  • 因此我们可以自定义sed分割符

获得匹配 (&)

  • 对于匹配到的字符串, 如果希望在原字符串的基础上进行修改
  • 那么可以使用&进行匹配字符串的获取

匹配组 (\1, \2 ..)

  • 匹配字符串组, 需要通过正则表达式的\(\)进行
  • 不同组别, 使用下标进行引用, 下标从1开始

字符串特殊标记 (\I)

  • 替换字符串可以使用一些特殊标记
  • 这些标记仅在GNU版本的sed中生效
  • 仅作参考
