引言:

linux的用户管理设计十分巧妙, 每个命令分工协作, 有条不紊地维护着系统工作,
本文将对常用命令选项选项进行讲解, 并补充一些常被混淆的知识点.

前置知识

系统基本信息

  • OS: CentOS 6.7 (Final)
  • Kernel: i686 Linux 2.6.32-573.el6.i686
  • Shell: bash 4.1.2

主机/终端

  • 资源总是有限的, 使用者普遍希望提高资源的利用率. 正因如此在CPU计算资源特别缺乏的计算机开荒时代,
  • 为了使造价高昂且稀缺的计算资源得到充分利用, 计算机被划分为了两个部分主机和终端
  • 主机是计算机的核心部件, 它包含着CPU与内存在内的硬件资源, 运行着一个操作系统
  • 终端一般由键盘和屏幕组成的输入输入设备, 通过网络终端可以和主机进行连接, 进而用上主机的计算存储资源

多用户系统(multi-user OS)

  • 主机/终端的结构, 决定了在主机端上的操作系统, 必须实现多用户的接入, 这个功能就是现在的多用户操作系统
  • 在实操上, 多个用户可以通过终端使用同一台计算机, 执行所需操作
  • 在本质上, 系统为每一位终端用户提供固定的时间片, 并照预设时间片轮流为用户服务

用户管理

用户基本知识

  • 用户也被称为账户, 它可以由数字和字母字符组成, 最小长度为1个字符, 最大长度为32字符

  • 用户可分为系统管理员,系统用户和一般用户

  • 系统管理员用户: 也称为root, 具有一切的操作权限

  • 系统用户: 与一般用户做区分, 一般被系统服务所使用, 软件安装时自动创建居多, 无登录shell居多

  • 一般用户: 一般是真实的用户(人), 用户主动创建居多, 一般设置登录shell

  • 用户名主要是便于人们记忆, 而操作系统更为敏感, 因此用户ID(UID)才是操作系统最关心的标识符

  • 一个用户名会对应一个用户ID, 用户ID有特殊的含义

    id 范围 用户
    0 系统管理员(root)
    1 ~ 499 保留系统使用
    500 ~ 65535 一般用户

用户组管理

用户组基本知识

  • 默认情况下, 系统会为创建的用户同时创建同名的用户组(主组)
  • 用户组可分为主组和从属组(补充组)
  • 主组: 用户的默认组，用户的gid所标识的组
  • 从属组: 用户添加到的组别

补充延伸

useradd vs adduser

  • useradd: 是系统编译好的二进制文件
  • adduser:
    • Debian/Ubuntu: 是一个perl脚本, 其后端调用的是useradd
    • RedHat/CentOS: 是一个指向useradd的连接文件
  • 作用: 二者均可实现用户的创建,
  • 对比: useradd属于低级的系统命令, 自定义性强; (ubuntu)adduser提供交互式用户创建, 人性化十足

用户批量创建 newusers/chpasswd

  • 用户的批量下载, 指的是一次性实现多个用户的创建
  • 传统的思路是依靠useradd和passsd通过编辑脚本的方式实现, 需要使用者对shell编程有所了解
  • 如果没有shell 编程经验, 那么newusers/chpasswd是很好的工具
  • newusers
    • 功能: 创建批量用户
    • 工作方式: 通过读取文件(格式与/etc/passwd相同), 批量创建出用户
  • chpasswd
    • 功能: 批量修改用户密码
    • 工作方式: 通过读取文件(格式: 用户名:口令,回车分割), 批量修改用户密码
