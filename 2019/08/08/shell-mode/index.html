shell的配置文件加载顺序

引言:

如果说bash有什么令人初学者难以理解的, 配置文件加载顺序纠纷问题, 必须占据一个席位.

前置知识

系统基本信息

  • OS: CentOS 6.7 (Final)
  • Kernel: i686 Linux 2.6.32-573.el6.i686
  • Shell: bash 4.1.2

辨清shell和bash

  • 什么是shell

    • shell: 是用户和操作系统沟通的桥梁.
    • shell是用户接入操作系统的接口, 只有文本的用户接口称为(CLI), 拥有图形的用户接口称为(GUI). 一个现代完整的操作系统主要由系统内核与应用程序两部分组成. 系统内核承担: 进程管理, 内存管理, I/O设备管理 以及 系统调用 的责任. 应用程序满足用户操作计算机完成各项任务的需求.
    • 为了使系统内核能顺利完成任务, 操作系统划分出了内核空间与用户空间, 前者对应系统内核, 后者对应应用程序. 计算机最根本的功能就是运算, 从内存中取出数据, 传递到CPU, 经过CPU加工运算出结果, 通过I/O总线按照需求输出. 在这个过程中, 用户有权利提交运算任务到操作系统上, 但无权告诉硬件 数据存放到内存的地址编号, CPU哪一个时刻进行调度, 选择使用哪一条I/O总线进行数据传输, 因为这些操作都被封装到内核空间这个黑盒子中.
      shell正是用户空间与内核空间沟通的隧道. 无论是CLI键入应用命令, 或是GUI的鼠标的操作, 无不例外是将用户得输入从用户空间通过相关得库(如果有), 传递给系统调用, 接着到达内核空间, 经硬件加工, 返回运算结果, 输出到指定目标.

  • 什么是unix shell

    • unix shell: 既是一个命令行解释器(CLI), 同时是一门编程语言.

    • 作为命令行解释器, unix shell提供了丰富的GNU系统工具, 作为一门编程语言, unix shell允许多个GNU系统工具结合使用. 用户通常使用终端模拟器与unxi shell进行交互, 较为著名的终端模拟器有: putty,Secure CRT和xshll等. unix shell普遍提供如文件通配符, 管道符, here文档, 命令替换, 变量, 流程控制, 状态测试 以及 迭代等功能.

    • 文件通配符 ( *.py, file?.py, file[13].py)

    • 管道符 (|)

    • here文档 (cat << EOF)

    • 命令替换 (`` / $())

    • 变量 (name=’hello’)

    • 流程控制 (for..do..done)

    • 状态测试 ([[]]/test)

    • 迭代

  • 什么是bash

    • bash的全称是(Bourne-Again SHell), 这个名字十分生动, 说明了它是Bourne SHell的继任者, 而Bourne SHell是一款类unix shell的产品, 作为命令交互接口.
    • 事实上, shell的种类有不少, 其中主流分之有三支Thompson shell -- Ken Thompson, Bourne shell -- Stephen Bourne 和 C shell. 因为开发理念不同, 这些shell呈现出不同的姿态, 降低资源占用的dash, 提倡友好交互的fish, 为超越Bourne-shell而生的bash, 现代化具有开创性的csh/tcsh.
    • shell的世界百花齐放, 但唯独bash坐上了Linux和macOS的快车, 成为了系统默认shell. 名气居高不下, 因此不少学习linux的不少初学者, 甚至会误认为bash就是shell.

正文内容

bash配置文件分类

  • 全局配置文件
    • /etc/profile

      用户用于设置环境变量的地方, 由于用户可以拥有多个shell, 但环境变量需要存储到同一个地方中. 特殊的地方在与该文件,对所有的shell都适用(zsh, tcsh, bash), 是用户首次登录时执行的文件, 用于获得系统信息

    • /ect/profile.d/*.sh

      shell启动的过程中, 系统的一些软件, 通过执行脚本, 进行初始化

    • /etc/bashrc

      bash的专属的启动执行文件, rc表示(run commands), rc后缀常被用于含有初始化命令的文件中, 初始化的内容包括bash命令的补齐, bash别名, bash颜色编码等

    • 全局配置系统启动bash时加载配置, 一般情况下不作修改.
  • 个人配置文件
    • ~/.bash_profile

      专属个人的环境变量设置, 与/etc/profile相对应

    • ~/.bashrc

      专属个人的bash设置, 每当登录和启动一个新的shell的时候将会执行, 与/etc/bashrc相对应

    • 个性化配置文件, 可添加alias配置, 自定义函数, 自定义命令提示符, 应保存于此, 可随意修改

交互/非交互

  • 交互型shell

    • 模式特点: shell等待用户的输入, 执行用户键入的命令, 返回执行结果至指定地方(默认打印到当前屏幕), 继续等待用户输入, 直至用户退出.
    • 常见场景: 通过ssh登录至某台服务器

  • 非交互型shell

    • 模式特点: shell通过读取存储在文件中的命令, 解析执行. 当读取到文件末尾得EOF时, 退出shell.
    • 常见场景: 使用sh script.sh执行脚本, fork出来的子shell

  • 查看当前shell – 交互/非交互

登录/非登录

  • 登录型shell

    • 模式特点: 需要用户名和密码才能进入shell
    • 常见场景: 启用--login选项生成的shell, 如su --login username切换用户

  • 非登录型shell

    • 模式特点: 不需要用户名和密码即可进入shell
    • 常见场景: 使用bash打开新的shell, GUI界面下打开”终端”(terminal), su username 切换用户时

  • 查看当前shell – 登录/非登录

不同场景下执行顺序

  • 登录/交互shell

    1. /etc/profile
    2. /etc/profile.d/*.sh
    3. /etc/bashrc
    4. ~/.bash_profile
    5. ~/.bashrc

  • 登录/非交互shell

    1. /etc/profile
    2. /etc/profile.d/*.sh
    3. /etc/bashrc
    4. ~/.bashrc

  • 非登录shell

    1. /etc/bashrc
    2. /etc/profile.d/*.sh
    3. ~/.bashrc

总结关系图

  • 序号同时表明运行顺序, 但实际运行过程, 往往并是非线性. 如/etc/profile脚本执行后, 将会去调度/etc/profile.d/*.sh进行执行. 当/etc/profile.d/*.sh运行结束后, /etc/profile才会执行

补充延伸

“${-}”含义

  • ${-}/$- 是bash的系统变量, 作用在于显示出当前shell的flag设定参数.

  • 查询flag

    • 查询结果解析

      • h (hashall):

        令bash记住通过PATH查询到的命令, 因此我们输入重复命令无需再查PATH

      • i (interactive):

        令shell可以进行交互, 因此我们能够键入输出返回结果

      • m (monitor):

        令shell可以控制jobs(作业), 因此我们能将作业放到后台

      • B (brace expand):

        令shell支持括号{}扩展, 因此我们能使用 mkdir file{1, 2}

      • H (history expand):

        令shell支持历史记录, 因此我们可以通过上下方向按键翻动历史命令, !#(history-number)执行命令

    • 可能出现查询结果

      • s :

        -s如果存在-s选项或者在处理选项后没有参数，则从标准输入读取命令, 因此在调用交互式shell时可设置位置参数。

      • e (errexit):

        当输入的命令错误时, 立刻退出bash

      • C (noclobber):

        禁止使用重定向来覆盖文件
