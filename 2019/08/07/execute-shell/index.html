bash脚本执行方式

发表于 2019-08-07 更新于 2019-09-16 分类于 linux ， script ， application
本文字数： 4.6k 阅读时长 ≈ 8 分钟

引言:

bash脚本有三类公认的执行方式, ./script.sh, sh script.sh以及source(.) script.sh. 通常情况下, 三种执行方式所得结果是相同的, 但也有例外的情况.

前置知识

系统基本信息

  • OS: Ubuntu 18.04 bionic
  • Kernel: x86_64 Linux 4.15.0-55-generic
  • Shell: bash 4.4.19
  • DE: Unity 7.5.0

bash 知识

  • shell进程类型

    • 父/子shell
      当启动一个termineral的时候, 我们将会得到一个shell. 帮助我们fork出这个shell的是一个父进程 . 当我们在当前shell, 启用一个应用时, 当前shell将会去fork一个子shell, 让子shell来完成任务, 对子shell而言, 当前shell是它得父shell. 当任务结束时, 子shell将被销毁, 返回到父shell中.

  • bash的变量类型

    • 本地变量
      生效范围: 当前shell进程

    • 环境变量
      生效范围: 当前shell以及子shell进程

    • 局部变量
      生效范围: 当前shell进程中的某个代码片段

    • 位置变量
      脚本代码中, 通过命令行传递参数时

    • 特殊变量
      bash解析的特殊变量

./script.sh (目录执行)

目录执行shell脚本, 可再细分为,工作目录执行和绝对路径执行.

  • 工作前提: 需为文件赋予x(执行) 权限

  • 工作方式: fork一个子shell进行脚本的执行

  • 工作目录执行: 是我们最常用的方式, 通过cd命令, 切换到存放script.sh文件的目录(该目录将被称为工作目录).

  • 绝对路径执行: 使用从(/)根目录开始得路径, 执行脚本, 定时任务中使用较多

  • 脚本运行结果参考

sh script.sh (sh 执行)

  • 工作前提: 仅需要使用者有读(r)的权限

  • 工作方式: fork一个子shell(sh)来执行脚本

  • 脚本运行结果参考

  • 注意事项

    本执行方式, 可分解为sh应用程序, 读取执行script.sh脚本文件. 因此sh命令的指向尤其重要.
    对于Ubuntu(16.04, 18.04)而言, 由于sh的链接是指向dash命令的. 当运行dash中不存在的命令, 脚本将会报错(echo {0..9} 打印出0-9字符). 这也就解析了, 我们时常会发现在CentOS中使用 sh执行正常的脚本, 拿到Ubuntu上, 同样使用sh却出现了报错的原因.

source(.) script.sh (shell环境执行)

  • 工作前提: 仅需要使用者有读(r)的权限

  • 工作方式: 在当前shell中执行脚本

补充延伸

dash

  • 什么是dash

    Dash的全称是(Debian Almquist shell), Dash是一个shell，当用作交互式登录shell时， 根据现代标准它的可用性很差，但在执行POSIX兼容脚本的速度比Bash快. 在Ubuntu 6.10后, 默认的shell变成了dash, 显然dash是Debian系列系统的特有工具. dash的设计目的使用尽可能少的代码来实现/bin/sh的功能.

  • dash的优点

    • 加速系统的启动时间, 这是因为dash启动比bash要迅速, 而shell在系统启动过程中占用了不少得时间
    • 占用更少的内存资源, dash只有100K大小, bash有900k的大小
    • 依赖更少的资源库(lib)
    • 在发生升级错误或磁盘故障时, dash被认为更可靠

  • dash vs bash

    • bash 拥有交互shell的众多良好功能, 并且作为系统的默认shell. dash主要用于非交互登录shell执行脚本
    • 一些bash脚本有set -k, 在dash中无法使用
    • bash面向的是人为用户, dash主要用作执行非交互脚本
    • dash仅有100k大小, bash有900k大小
    • 和bash相比, dash更适合用作快速系统启动的默认shell和脚本执行器
